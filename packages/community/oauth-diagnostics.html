Step 2: Add to Google Cloud Console

  1. Go to Google Cloud Console - Credentials
  2. Click on your OAuth 2.0 Client ID (e.g., "Engram Extension")
  3. Under "Authorized redirect URIs", add:
    Click "Get OAuth Redirect URL" above
  4. Also ensure this Supabase callback URL is present:
    https://wtylimoxssbgurawreps.supabase.co/auth/v1/callback
  5. Click Save

Step 3: Test Google Sign-In

  1. Wait 1-2 minutes for Google's changes to propagate
  2. Reload your Engram extension in chrome://extensions
  3. Open Engram side panel on ChatGPT/Claude/Perplexity
  4. Click "Sign in with Google"
  5. Complete the OAuth flow

Troubleshooting

If button doesn't work:

  1. Open DevTools Console (F12)
  2. Run this command:
    chrome.identity.getRedirectURL()
  3. Copy the output and use it in Step 2 above

Different redirect URLs for dev vs prod:

Each extension build (dev/prod) gets a different extension ID when loaded, which means different redirect URLs. You need to add BOTH to Google Cloud Console:
